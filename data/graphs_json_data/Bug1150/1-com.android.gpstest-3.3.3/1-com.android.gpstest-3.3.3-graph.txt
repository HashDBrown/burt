•  Acc - Estimated horizontal accuracy of the location (radius in meters of 68% confidence). In other words, if you draw a circle centered at this location's latitude and longitude, and with a radius equal to the accuracy, then there is a 68% probability that the true location is inside the circle.
•  H/V Acc - Estimated horizontal and vertical accuracy of the location, both at the 68% confidence. Horizontal accuracy is defined the same as Acc field above. Vertical accuracy is available on Android 8.0 Oreo and higher. Vertical accuracy example - if Alt is 150, and V Acc is 20 then there is a 68% probability of the true altitude being between 130 and 170 meters.
•  Speed - Speed, in meters/second over ground
•  S. Acc - Estimated speed accuracy of this location at 68% confidence, in meters per second. For example, if Speed is 5, and Speed Acc is 1, then there is a 68% probability of the true speed being between 4 and 6 meters per second.
•  Bearing - The horizontal direction of travel of the device (in degrees 0-360), and is not related to the device orientation.
•  B. Acc - Estimated bearing accuracy of this location at 68% confidence, in degrees. For example, if Bearing is 60, and Bearing Acc is 10, then there is a 68% probability of the true bearing being between 50 and 70 degrees.
•  # Sats - # satellites used in fix / # satellites in view
•  PDOP - Position (3D) dilution of precision (requires NMEA support)
•  H/V DOP - Horizontal / vertical dilution of precision (requires NMEA support)
•  TTFF - Time to first fix
•  ID or PRN - identifier of this satellite (pseudorandom noise for GPS)
•  CF - Label (e.g., L1) for carrier frequency of signal being tracked. If an unknown frequency, it shows the raw frequency in MHz (available on Android 8.0+, but not supported on all devices). The new GLONASS CDMA signal for L1 is shown as L1-C.
•  SNR - Signal-to-noise ratio (Android M and lower) (http://bit.ly/SNR-CN0)
•  C/N0 - Carrier-to-noise density (Android N and higher) (http://bit.ly/SNR-CN0)
•  Flags - (E) if the device has ephemeris data for the satellite, (A) if the device has almanac data for the satellite, (U) if the satellite was used in calculation of the most recent position fix. For SBAS, the Android spec does not specify if (U) refers to pseudoranges from the satellite being used in the position solution or to correction information originating from that satellite being used.
