Bug summary:

App will crash when user tries to add previous comment on a weight record

Observed Behavior (a description of what happened with the app):

The app will crash if the user tries to perform "Add previous comment" action on an existing entry whose comment section has already been edited.

Expected Behavior (a description of what you expected to happen):

User should be able to perform the "Add previous comment" action on an existing entry no matter its comment section has been edited or not.

Steps to Reproduce (the steps to reproduce the problem):

1. Input current weight and click the "Set Current Weight in kg" button.

2. Click on the entry to see the "Edit Weight" alert view.

3. Input something in comment section, and click "Go" button to close the alert view.

4. Long press the entry to see the action menu.

5. Click "Add previous comment" option.

6. App crushes.

Additional information:

Even if the comment section is not edited, but the "Edit Weight" view has popped up once, the app will still crash when the user performs the above steps.
