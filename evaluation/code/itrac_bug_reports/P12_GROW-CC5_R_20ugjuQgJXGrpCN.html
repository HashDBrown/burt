Bug summary:

App will crash if user tries to add new additive for a plant entry

Observed Behavior (a description of what happened with the app):

If the user tries to add a new additive for an existing plant entry, the app will crash

Expected Behavior (a description of what you expected to happen):

The app should allow users to add a new additive for an existing plant.

Steps to Reproduce (the steps to reproduce the problem):

1. Click the red plus button to create a new plant entry.

2. Input plant name, plant strain, and medium details.

3. Click the tick button to save the new plant entry.

4. Click on the entry to enter the entry detail screen.

5. Click on the "Feed" tab to enter the Feed information screen

6. Click on the "New additive" section to see the Additive alert view

7. Click on the OK button to save the new additive

8. App will crash
