Bug summary:

App will crash if user tries to add a new time range for a task entry

Observed Behavior (a description of what happened with the app):

The app will crash when a user tries to add a new time range to an existing task entry.

Expected Behavior (a description of what you expected to happen):

The user should be able to perform "add a new time range action" on an existing task entry.

Steps to Reproduce (the steps to reproduce the problem):

1. User clicks the three-dot button to see action list

2. User chooses the "Add task" option

3. User inputs task name and click "Add task" button to create a new entry

4. User long-press the entry to see the option list

5. User chooses the "Show time" option

6. User clicks the three-dot button to see the action list

7. User chooses the "Add a new time range" action

8. App crashes and exits
