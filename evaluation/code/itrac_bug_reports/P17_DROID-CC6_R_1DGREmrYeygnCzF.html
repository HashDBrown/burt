Bug summary:

"Add previous comment" crashes app if previous comments exist.

Observed Behavior (a description of what happened with the app):

On the main screen, if a comment has been made on any weight entry, after long pressing on a weight entry and selecting "Add previous comment," the app crashes.

Expected Behavior (a description of what you expected to happen):

An interface should have opened to add to previous comments to the selected entry.

Steps to Reproduce (the steps to reproduce the problem):

1. Open the app.

2. Specify a weight in the Weight field and press "Set Current Weight in kg" to create an entry.

3. Click on the entry to open the "Edit Weight" menu and type something in the Comment field. Close the "Edit Weight" menu.

4. Long press on an entry, then select "Add previous comment." The app crashes.

Additional information:

The nature of what "Add previous comment" should do is unclear: I am unsure of exactly what the expected behavior should be. When the option is selected before any comments have been added to any entries, an interface appears but states that there are no comments yet. I presume that this means that the option is meant to allow users to copy the text of comments made on previous entries to new ones, but I am unsure of this.
