Bug summary:

When the user moves the selected time to a certain task, the time disappears

Observed Behavior (a description of what happened with the app):

When selecting a certain time to move to a task, the task disappears from the task time window.

Expected Behavior (a description of what you expected to happen):

The expected behavior is that the time slot will still appear in the selected window after moving it to Task1 since it is already in Task1

Steps to Reproduce (the steps to reproduce the problem):

1. Open the application

2. Create a new task

3. Long click the task

4. Select "Show Times"

5. Select a time frame

6. Update the time frame

7. Long click the time frame

8. Select "Move Time"

9. Move to the current task

10. The task will disappear from the list

Additional information:

By leaving and returning to the task, the time frame does appear in the list.
