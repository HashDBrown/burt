Bug summary:

the "move time" operation makes a created task disappear

Observed Behavior (a description of what happened with the app):

Everything works as expected creating a task within the app, but when I attempt to perform the "move task" operation on an existing task, then the task disappears from the task list.

Expected Behavior (a description of what you expected to happen):

Instead of making the task disappear, the move task button should allow the user to modify the start and end time of the task.

Steps to Reproduce (the steps to reproduce the problem):

1. From the home page, press the menu button on the top right of the screen

2. select the "add a new time range option", and create any valid time range

3. long click the newly created task that appears on the main page

4. select the move time button from the menu that appears

5. click the button corresponding to the name of the task that you created

6. At this point the screen returns to the main menu and the task you selected is no where to be found
