Bug summary:

The app will crash after a series of operations

Observed Behavior (a description of what happened with the app):

The app crashes

Expected Behavior (a description of what you expected to happen):

The app should behave correctly instead of crashing.

Steps to Reproduce (the steps to reproduce the problem):

1. Click the app icon to start the app.

2. An introduction will shown on the screen, click the "OK" button to close this introduction.

3. Click the "Action overflow" button on the right-top corner, and then add a task.

4. Type the task name, such as "test", and click the "ADD TASK" button.

5. Now, we should be able to see a task named "test" that has been listed.

6. Perform a long click on this created task. Then, on the popped menu, select "Show Times", nothing happened.

7. Click the "Action overflow" button again to add a new time range. Now, the app will crash.

Additional information:

This bug should be assigned the highest priority to be fixed as it crashes after some operations.
