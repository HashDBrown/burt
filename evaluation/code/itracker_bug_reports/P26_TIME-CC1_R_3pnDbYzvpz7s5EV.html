Bug summary:

Moving a time range for a specific task to itself causes the time range to disappear from the UI

Observed Behavior (a description of what happened with the app):

When creating a time range for a task and then attempting to move said time range to the same task, the time range disappears from the UI.

Expected Behavior (a description of what you expected to happen):

The time range should not disappear when attempting to move it to the same task you created it for.

Steps to Reproduce (the steps to reproduce the problem):

1. Dismiss the popup explaining the app

2. Click the three dot option bottom in the top right corner

3. From the dropdown, select "Add Task"

4. Give the task a name and hit "Add Task"

5. Long click on the newly created task and select "Show Times" from the dropdown

6. Click the three dot option bottom in the top right corner

7. Click the button for creating a new time range from the dropdown

8. Create a new time range and scroll to the bottom and select "Ok"

9. Long click on the newly created time range and select "Move Time" from the dropdown

10. From the popup select the name of the task you created before
