Bug summary:

App crashes when you select add previous comment, but there is only one weight entry

Observed Behavior (a description of what happened with the app):

After tapping and dragging a single weight entry, I was presented with two options: add previous comment and delete measure. Upon selecting add previous comment, the app crashed.

Expected Behavior (a description of what you expected to happen):

The app shouldn't crash when I select add previous comment

Steps to Reproduce (the steps to reproduce the problem):

1. Open the app

2. Tap the white weight text input field

3. Type 23

4. Tap 'Go'

5. Tap 'Set Current Weight in kg'

6. Tap on the newly created entry

7. Add a comment "comment"

8. Tap 'Go'

9. Tap and hold on the entry

10. Tap 'Add previous comment' on the pop up

11. The app crashes

Additional information:

This bug was encountered from a cold start. There were no other weight entries. Selecting delete works as intended.
